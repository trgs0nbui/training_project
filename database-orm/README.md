# Tìm hiểu về Database & ORM
## 1. Models
- Một model là duy nhất và định nghĩa nguồn thông tin nhất định về dữ liệu của nó. Nó chứa các trường cần thiết và các hành vi của dữ liệu mà nó đang lưu trữ
- Mọi model là 1 python class và là class con của `django.db.models.Model`
- Mỗi thuộc tính của model đều biểu diễn trường trong cơ sở dữ liệu
- Ta có ví dụ dưới đây
```
    from django.db import models

    class Person(models.Model):
        first_name = models.CharField(max_length=30)
        last_name = models.CharField(max_length=30)
```
- Class trên tương đương với một bảng trong cơ sở dữ liệu với minh họa như sau:
```
    CREATE TABLE myapp_person (
        "id" bigint NOT NULL PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
        "first_name" varchar(30) NOT NULL,
        "last_name" varchar(30) NOT NULL
    );
```

### a. Sử dụng Models trong Django
- Khi định nghĩa các model, ta cần nói cho Django biết rằng ta đang sử dụng các model đó. 
- Để làm được như vậy, ta cần chỉnh sửa `settings` file, cụ thể hơn là thay đổi `INSTALLED_APPS` để thêm tên của module chứa `models.py`

```
    INSTALLED_APPS = [
        #...
        "myapp",
        #...
    ]
```
- Sau khi đã cho Django biết những model cần sử dụng, cần đảm bảo rằng lệnh `manage.py migrate` được chạy

### b. Automatic primary key fields
- Django tự động đưa vào mỗi model 1 primary key tự động tăng 
### Fields
- 1 field name trong Django không thể là 1 Python key words ví dụ: pass, str...
- 1 field name không thể chứa nhiều hơn 1 dấu gạch dưới, dựa vào cách mà Django truy vấn cú pháp 
- 1 field name không thể kết thúc bằng dấu gạch dưới
- 1 field name không thể `check` vì nó sẽ ghi đề phương thức `Model.check()`

### c. Cẩn thận khi sử dụng `related_name` và `related_query_name`
- Nếu sử dụng reverse relation ship, bắt buộc phải xác định 1 tên đảo ngược và tên truy vấn duy nhất cho trường đó.
-> Nó có thể gây ra 1 vấn để ở các abstract cơ bản bởi vì các trường của class này bao gồm các class con cùng với chính những giá trị của các thuộc tính
- Để xử lý vấn đề này, khi dùng reverse relationship trong 1 abstract class, phần giá nên chứa `'%(app_label)s'` và `'%(class)s'`
```
    from django.db import models

    class Base(models.Model):
        m2m = models.ManyToManyField(
            OtherModel,
            related_name="%(app_label)s_%(class)s_related",
            related_query_name="%(app_label)s_%(class)ss",
        )

        class Meta:
            abstract = True

    class ChildA(Base):
        pass

    class ChildB(Base):
        pass
```
